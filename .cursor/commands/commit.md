# Commit and push our work

EXECUTE MODE
Fetch changes from the remote repository.
RESEARCH MODE
THAN Open all staged files and read them to understand what was done.
Check all files to have only safe information: no keys, passwords and sesetive data allowed to send to githup. k8s services use secrets synced from Vault via ESO — never commit `.env` files, `k8s/secret.yaml` files, or any file containing raw credentials.
EXECUTE MODE
Create commit message and commit it.
Push to the repository.
